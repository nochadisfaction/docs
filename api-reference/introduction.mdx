The Pixelated Empathy API provides programmatic access to the platform's core capabilities — bias analysis, training session management, competency analytics, and crisis detection. Use the API to build custom integrations, automate workflows, and embed mental health training intelligence into your existing systems.

## Base URL

All API requests are made to the following base URL:

```
https://api.pixelatedempathy.com/v1
```

<Note>
  Always use HTTPS. HTTP requests are rejected with a `301` redirect. All data in transit is encrypted with TLS 1.3.
</Note>

## API versioning

The API uses URL-based versioning. The current stable version is **v1**.

| Version | Status | End-of-life |
|---|---|---|
| `v1` | **Current** — actively maintained | No planned EOL |
| `v0` (beta) | Deprecated | June 30, 2025 |

When a new version is released, the previous version remains supported for at least 12 months. Breaking changes are never introduced within a version — only additive, backwards-compatible changes.

Version is included in the URL path:

```
https://api.pixelatedempathy.com/v1/sessions
https://api.pixelatedempathy.com/v1/bias-analysis/analyze
```

## Authentication

The API supports two authentication methods:

<CardGroup cols={2}>
  <Card title="JWT bearer tokens" icon="key">

  <Card title="API keys" icon="lock">

See the [Authentication guide](/api-reference/authentication) for detailed setup instructions, token refresh flows, and role-based access control.

## Rate limits

Rate limits vary by authentication role and endpoint category:

| Role | Standard endpoints | Analytics endpoints | Batch endpoints |
|---|---|---|---|
| **Admin** | 1,000 req/min | 500 req/min | 100 req/min |
| **Supervisor** | 500 req/min | 250 req/min | 50 req/min |
| **Therapist** | 200 req/min | 100 req/min | 20 req/min |
| **User** | 60 req/min | 30 req/min | 10 req/min |
| **API key (service)** | 1,000 req/min | 500 req/min | 100 req/min |

Rate limit status is returned in response headers:

```
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 997
X-RateLimit-Reset: 1700000060
```

<Warning>
  Exceeding rate limits returns a `429 Too Many Requests` response. Implement exponential backoff in your client to handle rate limiting gracefully.
</Warning>

## Response format

All responses are returned in JSON format with a consistent envelope structure.

### Successful response

```json
{
  "status": "success",
  "data": {
    "id": "sess_abc123",
    "type": "training_session",
    "attributes": {
      "scenario": "intake-assessment",
      "duration_minutes": 28,
      "competency_score": 82
    }
  },
  "meta": {
    "request_id": "req_7f3a9b2c",
    "timestamp": "2025-01-15T10:30:00Z"
  }
}
```

### Error response

```json
{
  "status": "error",
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "The 'text' field is required for bias analysis.",
    "details": [
      {
        "field": "text",
        "issue": "required",
        "message": "This field cannot be empty."
      }
    ]
  },
  "meta": {
    "request_id": "req_8d4b1e3f",
    "timestamp": "2025-01-15T10:30:01Z"
  }
}
```

## Error codes

The API uses standard HTTP status codes combined with application-specific error codes:

| HTTP status | Error code | Description |
|---|---|---|
| `400` | `VALIDATION_ERROR` | Request body or parameters failed validation |
| `400` | `INVALID_FORMAT` | Request body is not valid JSON |
| `401` | `UNAUTHORIZED` | Missing or invalid authentication credentials |
| `401` | `TOKEN_EXPIRED` | JWT token has expired; refresh required |
| `403` | `FORBIDDEN` | Authenticated user lacks permission for this resource |
| `403` | `ROLE_INSUFFICIENT` | User's role does not have access to this endpoint |
| `404` | `NOT_FOUND` | Requested resource does not exist |
| `409` | `CONFLICT` | Resource state conflict (e.g., duplicate creation) |
| `422` | `UNPROCESSABLE` | Request is valid JSON but semantically incorrect |
| `429` | `RATE_LIMITED` | Rate limit exceeded |
| `500` | `INTERNAL_ERROR` | Unexpected server error |
| `502` | `SERVICE_UNAVAILABLE` | Upstream service temporarily unavailable |
| `503` | `MAINTENANCE` | API is undergoing scheduled maintenance |

## Pagination

List endpoints return paginated results using cursor-based pagination for consistent performance across large datasets.

### Request parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `limit` | integer | 20 | Number of results per page (max 100) |
| `cursor` | string | — | Cursor from the previous response's `next_cursor` field |
| `sort` | string | `created_at` | Field to sort by |
| `order` | string | `desc` | Sort direction: `asc` or `desc` |

### Paginated response

```json
{
  "status": "success",
  "data": [
    { "id": "sess_001", "created_at": "2025-01-15T10:00:00Z" },
    { "id": "sess_002", "created_at": "2025-01-14T09:30:00Z" }
  ],
  "pagination": {
    "has_more": true,
    "next_cursor": "eyJpZCI6InNlc3NfMDAyIn0=",
    "total_count": 156
  },
  "meta": {
    "request_id": "req_a1b2c3d4",
    "timestamp": "2025-01-15T10:30:00Z"
  }
}
```

To fetch the next page:

```bash
curl "https://api.pixelatedempathy.com/v1/sessions?limit=20&cursor=eyJpZCI6InNlc3NfMDAyIn0=" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

## HIPAA compliance

All API endpoints are HIPAA-compliant by default. Protected health information (PHI) is:

- Encrypted in transit (TLS 1.3) and at rest (AES-256)
- Never logged in plaintext in server logs
- Automatically purged according to your organization's data retention policy
- Accessible only to authenticated users with appropriate role permissions

<Info>
  A signed Business Associate Agreement (BAA) is required before transmitting PHI through the API. Contact [enterprise@pixelatedempathy.com](mailto:enterprise@pixelatedempathy.com) to initiate the BAA process.
</Info>
