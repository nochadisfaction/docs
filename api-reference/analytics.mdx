title: 'Analytics API'
description: 'Access dashboard data, performance metrics, competency scores, bias reduction trends, and export capabilities for research and compliance'
---

The Analytics API provides programmatic access to all performance and training data generated on Pixelated Empathy. Use these endpoints to build custom dashboards, power reporting workflows, track competency development over time, and export data for research and compliance purposes.

## Dashboard data

Retrieve a summary of key metrics for the authenticated user's dashboard.

### `GET /v1/analytics/dashboard`

<Note>
  **Required scope:** `analytics:read` · **Rate limit:** Analytics tier
</Note>

```bash
curl "https://api.pixelatedempathy.com/v1/analytics/dashboard?period=30d" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

### Query parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `period` | string | `30d` | Time period: `7d`, `30d`, `90d`, `6m`, `1y`, `all` |
| `user_id` | string | — | Target user (supervisors and admins only) |
| `group_id` | string | — | Target training group |

## Performance metrics over time

Retrieve detailed performance metrics across a specified time range, broken down by interval.

### `GET /v1/analytics/performance`

```bash
curl "https://api.pixelatedempathy.com/v1/analytics/performance?period=90d&interval=week&metric=overall_score" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

### Query parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `period` | string | `30d` | Time period: `7d`, `30d`, `90d`, `6m`, `1y` |
| `interval` | string | `week` | Data granularity: `session`, `day`, `week`, `month` |
| `metric` | string | `overall_score` | Metric to retrieve: `overall_score`, `empathic_accuracy`, `bias_awareness`, `crisis_response`, `cultural_sensitivity`, `session_duration`, `turn_count` |

## Comparative benchmarking

Compare a user's performance against reference groups.

### `GET /v1/analytics/benchmarks`

<Note>
  **Required scope:** `analytics:benchmarks` · **Available to:** Supervisors and admins
</Note>

```bash
curl "https://api.pixelatedempathy.com/v1/analytics/benchmarks?user_id=usr_k1l2m3n4&benchmarks=cohort,program,national" \
```

| Parameter | Type | Description |
|---|---|---|
| `user_id` | string | Target user to benchmark |
| `benchmarks` | string | Comma-separated benchmark types: `cohort`, `program`, `national`, `expert` |
| `period` | string | Time period for score calculation |

## Export endpoints

Export analytics data for research, compliance, and institutional reporting.

### `POST /v1/analytics/export`

<Warning>
  Export endpoints may generate large files and are rate-limited to the batch tier. De-identified exports exclude all personally identifiable information. Full exports require admin role and appropriate data use agreements.
</Warning>

```bash
curl -X POST https://api.pixelatedempathy.com/v1/analytics/export \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "export_type": "research",
    "format": "csv",
    "date_range": {
      "start": "2024-07-01T00:00:00Z",
      "end": "2025-01-15T23:59:59Z"
    },
    "include": [
      "session_metadata",
      "competency_scores",
      "bias_analysis",
      "session_frequency"
    ],
    "de_identify": true,
    "group_id": "grp_fall2024"
  }'
```

### Export types

<Tabs>
  <Tab title="Research export">
    De-identified dataset suitable for academic research. Removes all PII and assigns random participant IDs. Includes session-level and response-level metrics.

    ```json
    {
      "export_type": "research",
      "format": "csv",
      "de_identify": true
    }
    ```
  </Tab>
  <Tab title="Compliance export">
    Structured report for accreditation bodies and regulatory compliance. Maps competency scores to standard frameworks (APA, CACREP).

    ```json
    {
      "export_type": "compliance",
      "format": "pdf",
      "framework": "cacrep",
      "include_attestations": true
    }
    ```
  </Tab>
  <Tab title="Institutional report">
    Aggregate analytics for institutional dashboards and program review. Includes trends, benchmarks, and ROI metrics.

    ```json
    {
      "export_type": "institutional",
      "format": "json",
      "aggregate_level": "program"
    }
    ```
  </Tab>
</Tabs>

### Export response

Exports are processed asynchronously. The initial response includes a job ID for status polling:

```json
{
  "status": "success",
  "data": {
    "export_id": "exp_x1y2z3a4",
    "status": "processing",
    "estimated_completion_seconds": 45,
    "poll_url": "https://api.pixelatedempathy.com/v1/analytics/export/exp_x1y2z3a4"
  }
}
```

### Check export status

```bash
curl https://api.pixelatedempathy.com/v1/analytics/export/exp_x1y2z3a4 \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

```json
{
  "status": "success",
  "data": {
    "export_id": "exp_x1y2z3a4",
    "status": "completed",
    "download_url": "https://exports.pixelatedempathy.com/exp_x1y2z3a4.csv",
```

<Info>
  Export download links are signed and expire after 7 days. Generate a new export if the link has expired. All export files are encrypted at rest and transmitted over TLS.
</Info>

## Error responses

| Status | Error code | Description |
|---|---|---|
| `400` | `INVALID_PERIOD` | Unrecognized time period value |
| `400` | `INVALID_METRIC` | Unrecognized metric name |
| `403` | `BENCHMARK_ACCESS_DENIED` | User role cannot access benchmarking data |
| `403` | `EXPORT_REQUIRES_ADMIN` | Full (non-de-identified) exports require admin role |
| `404` | `USER_NOT_FOUND` | Specified `user_id` does not exist or is not accessible |
| `404` | `EXPORT_NOT_FOUND` | Export job ID does not exist |
| `429` | `RATE_LIMITED` | Rate limit exceeded for analytics tier |
